# Provenance audit

Audit date: 2026-06-22. This report records provenance evidence present in the repository and its Git history, together with provenance statements supplied by the repository owner. It does not independently certify ownership or compare local files with upstream releases.

Confidence means:

- **HIGH** — the repository contains direct, mutually consistent evidence such as a source URL, named author/project and license text, or a continuous first-party development history.
- **MEDIUM** — the repository contains a source or license declaration, but lacks an exact upstream revision, independent provenance, or another part of the chain of custody.
- **LOW** — origin, author or applicable license cannot be established from repository evidence. Such entries remain high risk.

Confidence measures the strength of repository evidence, not legal certification. Licensing risk separately measures unresolved third-party ownership or redistribution questions. `UNKNOWN` is used whenever the available evidence does not prove the requested fact. A Git committer is not treated as the author or copyright holder of imported material merely because they committed it.

## First-party AI-assisted work

- Code or assets generated by AI agents under the repository owner’s direction are treated as first-party repository work when the owner identifies that creation path and no evidence indicates an external import.
- A missing external upstream URL is not a provenance defect for such work. The owner statement, Git history, local project attribution and absence of external source claims can together support **HIGH** provenance confidence.
- This classification is a repository provenance finding based on the owner’s representation, not a legal certification or a guarantee of originality.
- It does not apply to components that cite external upstreams, contain identified third-party code, ship compiled binaries without source correspondence, or have conflicting license evidence.

## Confidence summary

### HIGH confidence components

- Poker code and first-party ChatGPT-generated chip atlas/derived PNGs
- Catch the Cats
- 2048 and Canvas Tetris
- Breakout, Flappy Bird, Minesweeper, Pong, Snake, Asteroids, Space Invaders, Frogger, Galaga, Missile Command, Simon, Connect Four, Whac-A-Mole, Memory Match, Sokoban, Brick Breaker, Tic-Tac-Toe, Hangman, Solitaire and Sudoku as AI-assisted first-party implementations based on the owner’s representation, local attribution and Git history
- Poppins remote font dependency
- Postgres.js and ws

### MEDIUM confidence components

- Pacman and T-Rex Runner: named external upstreams and local license claims exist, but the applicable license for the unknown imported revision is unresolved.
- Freedoom browser game and data, and the Dwasm/PrBoomX build: projects and licenses are named, but exact releases/build correspondence are missing.
- Synthesized Web Audio effects, Klaro, Supabase JavaScript client and the remaining locked npm set.

### LOW confidence components

- Google libarchive/Comlink bundle: only a partial embedded header exists; exact upstream and complete JS/WASM license scope are unknown.
- Game catalogue SVG thumbnails and the portal OpenGraph SVG: addition commits exist, but author/source and explicit asset license are unknown.

### HIGH RISK components

Provenance confidence and licensing risk are separate. First-party AI-assisted work can have **HIGH** provenance confidence based on the owner’s representation without being legally certified. The remaining high-risk entries involve external imports or compiled third-party artifacts, not merely a missing upstream URL.

- Pacman and T-Rex Runner remain **INVESTIGATE** because the named upstream is known but the license for the exact imported revision is unresolved.
- Dwasm/PrBoomX compiled runtime remains **INVESTIGATE** because exact corresponding source and build provenance are absent.
- Google libarchive/Comlink bundle remains **INVESTIGATE** because exact upstream and complete JS/WASM license scope are unknown.

## Coverage notes

- Scanned game directories and pages, `about/licenses.html`, repository documentation, `third_party/`, notices, lockfiles, relevant comments/headers and Git history.
- Scanned tracked SVG, PNG, audio, font, WASM, `.data` and `.bin` files. No tracked audio or font binaries were found.
- Inline SVG UI icons are part of HTML/application source and have no separate provenance declarations; they therefore inherit the provenance status of their containing first-party file unless future evidence identifies a separate source.
- Hosted analytics/advertising scripts are external services, not copied game assets; they are documented in `docs/third-party-notices.md` and are outside this game/asset provenance table.
